# pocalab bug & experience survey (for friend testers)

Paste into Google Forms / Typeform. Keep it short — these are friends doing you a favor, not a paid panel. Aim for under 5 minutes to complete.

**Intro blurb (top of form):**
> Thanks for testing pocalab! Make a deck of cards like you actually would for printing — upload photos, crop them, set up fronts/backs, and download the PDF. Since these are getting printed, please flag anything that looks visually off, broken, or just feels wrong while you're using it.

## 1. Context
- **What device/browser did you use?** (short answer — e.g. "iPhone 14, Safari" or "Windows laptop, Chrome")
- **Had you used pocalab before today?** (Yes / No)

## 2. Using it — experience & bugs
- **Did everything work the way you expected at each step (uploading, cropping, adding backs, building the deck, downloading)? If not, what happened and at which step?** (open) — *main bug-catching question*
- **Did anything look broken, cramped, cut off, or overlapping on your screen — especially on mobile?** (open)
- **Did you try editing or re-cropping a card you'd already added?** (Yes / No) — if yes, **did it behave the way you expected?** (open)
- **Did you use a shared back (same back for multiple cards) or a different back per card?** (Shared / Per-card / Both) — **did it work the way you expected?** (open)
- **On a scale of 1–5, how smooth was the overall experience?** (1 = lots of friction, 5 = smooth)
- **Where did you get stuck or confused, if anywhere?** (open)

## 3. Wrap-up
- **What's missing that would make you actually use this for a real event (birthday, kpop event, etc.)?** (open)
- **Anything else — bugs, ideas, complaints?** (open, optional)

---

**Why these questions:** section 2 is built to surface concrete bugs and friction in the actual upload → crop → deck → export flow, including known trouble spots from earlier in-person testing (mobile layout cramping, shared-back behavior) — see `docs/ux-audit-fix-plan.md`. Section 3 keeps the two forward-looking questions that feed the business-page and marketing-spike ideas already in `docs/INBOX.md`.
