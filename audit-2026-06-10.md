# UX Audit Report — Photocard Generator

> **Date:** 2026-06-10 · **Reviewed:** `http://localhost:5175` (Vite dev server) + full source (`src/`) · **Tools run:** axe-core 4.11.4, pa11y (WCAG2AA), Lighthouse (performance/accessibility/best-practices), manual code review

---

## Executive summary

The app is well-structured and covers the essentials — progress labels, a disabled confirm button during export, low-res warnings, and a beforeunload guard for unsaved work. However, three issues need to be resolved before shipping: a silent discard of user work when cancelling mid-flow, a swallowed crop-export error, and a marginal WCAG AA contrast failure on the primary "Choose file" button. **3 blockers, 6 important, 4 polish.**

**Snapshot**

- Checklist items passing (applicable): **~22 / ~35**
- Core Web Vitals (dev-mode — *see note*): LCP **8.7 s** · INP **n/a** · CLS **0** *(2026 thresholds: <2.5s / <200ms / <0.1)*
- Lighthouse scores: Performance **62** · Accessibility **87** · Best Practices **100**
- Accessibility (axe + pa11y): **2** violations across **1** WCAG 2.2 criterion (color-contrast)

> **⚠ Performance note:** LCP 8.7 s and FCP 5.3 s are dev-mode numbers from an unminified, unoptimized Vite dev server. CLS (0) and TBT (0 ms) are reliable in any mode. Run `npm run build && npm run preview` and re-measure before treating LCP as a real problem.

## What passed

- **Unsaved-work guard** (`useBeforeUnload.ts`): fires correctly whenever the deck has cards or the user is mid-flow — a rare thing to see done right.
- **Crop-export loading state**: confirm button disables and shows "Exporting…" during the canvas operation; prevents double-submits (`CropEditor.tsx:79,272`).
- **Low-res warning**: visible, distinct alert when source image is below 300 DPI (`CropEditor.tsx:264`).
- **Back-image replacement confirmation**: `window.confirm` on `App.tsx:72` prevents accidental overwrites of a selected back image.
- **Zero CLS (0)** and **zero TBT (0 ms)**: no layout shift and no main-thread blocking — the core interaction model is solid.
- **Best-practices score: 100** (Lighthouse) — no deprecated APIs, HTTPS-ready, clean console.
- **Undo/redo history** in the crop editor is a genuine quality-of-life feature rarely seen in tools of this type.
- **Semantic HTML throughout**: real `<button>` elements, `aria-label` on the remove button, `aria-hidden` on decorative guide overlays.
- **Dark mode**: full `@media (prefers-color-scheme: dark)` implementation with CSS variables.
- **Input validation**: file type and size checked client-side with visible, inline error messages (`ImageUpload.tsx:15-26`).
